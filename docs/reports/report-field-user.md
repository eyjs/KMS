# KMS 현업 사용자 가이드

> **대상**: 문서 관리 담당자 / 현업 사용자
> **목적**: 시스템 이해 및 업무 가이드
> **버전**: 1.0 | 2026-02-08

---

## 1. 시스템 소개

### 이 시스템이 하는 일

```
문서를 체계적으로 관리합니다

- 문서 업로드 및 저장
- 분류 (보험사/상품/문서유형)
- 문서 간 연결 (관계 설정)
- 문서 상태 관리 (신규 → 활성 → 만료)
- 문서 미리보기
```

### 이 시스템이 안 하는 일

```
- PDF 내용 자동 분석 (외부 시스템 담당)
- 문서 내용 검색 (외부 시스템 담당)
- AI 챗봇 (외부 시스템 담당)
```

---

## 2. 허용 파일 형식

### 업로드 가능

| 형식 | 확장자 | 비고 |
|------|--------|------|
| **PDF** | .pdf | 표준 문서 형식 |
| **Markdown** | .md | 텍스트 기반 문서 |
| **CSV** | .csv | 데이터 표 형식 |

### 업로드 불가

| 형식 | 확장자 | 대안 |
|------|--------|------|
| Word | .doc, .docx | PDF로 변환 후 업로드 |
| PowerPoint | .ppt, .pptx | PDF로 변환 후 업로드 |
| Excel | .xls, .xlsx | CSV로 변환 후 업로드 |

> **왜 막나요?** Word/PPT는 형식이 복잡해서 뷰어가 불안정합니다. PDF/CSV로 변환하면 안정적으로 볼 수 있습니다.

## 3. 문서 업로드

### 업로드 전 체크리스트

```
[ ] 파일 형식 확인 (PDF, MD, CSV만 가능)
[ ] 파일명 확인 (보험사_상품_문서유형 형식 권장)
[ ] 중복 문서 확인 (같은 분류에 이미 있는지)
```

### 파일명 권장 형식

```
보험사_상품_문서유형_날짜.pdf

예시:
KB손해보험_든든어린이보험_상품요약본_202602.pdf
삼성화재_통합보험_판매스크립트_202601.pdf
```

> **왜 이렇게?** 시스템이 파일명에서 자동으로 분류를 제안합니다.

### 업로드 순서

```
1. [문서 업로드] 버튼 클릭
2. 파일 선택 (PDF/MD/CSV)
3. 분류 선택
   - 보험사 선택
   - 상품 선택
   - 문서유형 선택
4. [저장] 클릭
```

## 4. 분류 체계

### 분류 축 (3가지)

| 축 | 예시 | 설명 |
|----|------|------|
| **보험사** | KB손해보험, 삼성화재 | 어느 보험사의 문서인가 |
| **상품** | 든든 어린이보험, 통합보험 | 어떤 상품의 문서인가 |
| **문서유형** | 상품요약본, 판매스크립트 | 어떤 종류의 문서인가 |

### 문서유형 목록

| 문서유형 | 설명 | 변경 빈도 |
|---------|------|----------|
| 약관 | 보험 계약 조건 | 연간 |
| 상품설명서 | 상품 상세 설명 | 분기 |
| 상품요약본 | 상품 핵심 요약 | 분기 |
| 판매스크립트 | 영업 화법 | 분기 |
| 시책 | 영업 인센티브 | 수시 |
| 수수료표 | 수수료 정보 | 수시 |
| 교육자료 | 교육/연수 자료 | 연간 |

## 5. 문서 상태 (라이프사이클)

### 상태 흐름

```
DRAFT (초안)
   ↓
ACTIVE (활성) ← 실제 사용 중인 문서
   ↓
DEPRECATED (만료) ← 더 이상 사용하지 않음
```

### 상태 설명

| 상태 | 의미 | 색상 |
|------|------|------|
| DRAFT | 작성 중, 검토 필요 | 회색 |
| ACTIVE | 현재 사용 중 | 초록 |
| DEPRECATED | 만료됨, 참고용 | 빨강 |

### 상태 변경 시점

```
DRAFT → ACTIVE: 검토 완료, 사용 시작
ACTIVE → DEPRECATED: 새 버전 등록 또는 만료
```

## 6. 문서 연결 (관계)

### 관계 유형

| 관계 | 의미 | 예시 |
|------|------|------|
| **부모-자식** | 상위-하위 문서 | 약관 → 상품설명서 |
| **형제** | 같은 레벨 문서 | 상품요약본 ↔ 판매스크립트 |
| **참조** | 관련 문서 | 시책 → 수수료표 |
| **대체** | 버전 교체 | 구버전 → 신버전 |

### 관계 설정 순서

```
1. 문서 상세 페이지 진입
2. [관계 추가] 버튼 클릭
3. 관계 유형 선택
4. 연결할 문서 검색/선택
5. [저장] 클릭
```

### 관계 주의사항

```
- 순환 참조 불가 (A → B → C → A 는 불가)
- 자기 참조 불가 (A → A 는 불가)
- 중복 관계 불가 (같은 관계 두 번 추가 불가)
```

## 7. 문서 품질 검토

### 신선도 (Freshness)

| 상태 | 의미 | 조치 |
|------|------|------|
| HOT (뜨거움) | 최신 문서 | 유지 |
| WARM (따뜻함) | 곧 검토 필요 | 검토 예약 |
| COLD (차가움) | 오래된 문서 | 즉시 검토 |

### 문서유형별 검토 주기

| 문서유형 | 검토 주기 | 비고 |
|---------|----------|------|
| 시책, 수수료표 | 30일 | 자주 변경 |
| 상품설명서, 스크립트 | 90일 | 분기별 검토 |
| 약관, 교육자료 | 365일 | 연간 검토 |

## 8. 중복 문서 방지 (SSOT)

### 규칙

```
같은 분류 경로에 ACTIVE 문서는 1개만 가능

예:
KB손해보험 > 든든 어린이보험 > 상품요약본 → 1개만 ACTIVE
```

### 중복 발생 시

```
새 문서를 ACTIVE로 전환하면:
→ 기존 문서는 자동으로 DEPRECATED 처리
→ 두 문서 간 "대체" 관계 자동 생성
```

## 10. 자주 묻는 질문 (FAQ)

**Q: Word 파일을 업로드하고 싶어요**
> PDF로 변환해서 업로드하세요. Word에서 [파일] → [내보내기] → [PDF] 선택

**Q: 기존 문서를 수정하고 싶어요**
> 수정이 아닌 "새 버전 업로드"를 권장합니다. 기존 문서는 DEPRECATED, 새 문서는 ACTIVE로 설정.

**Q: 관계를 잘못 설정했어요**
> [관계 삭제] 버튼으로 삭제 후 다시 설정하세요.

**Q: 문서가 안 보여요**
> 상태가 DEPRECATED인지 확인하세요. 필터에서 "만료 문서 포함" 체크.

**Q: 검색이 안 돼요**
> 현재는 메타데이터 검색만 가능합니다. 문서 내용 검색은 추후 지원 예정.
