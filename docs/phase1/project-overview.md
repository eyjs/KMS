## 2. 핵심 개념: 프레임워크

### 2.1 왜 "프레임워크"인가?

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│   "보험업만 쓰는 시스템 아니에요.                                     │
│    어떤 업종이든, 어떤 문서든                                         │
│    '분류 → 연결 → 관리'가 필요하면 쓸 수 있어요."                     │
│                                                                     │
│   ┌───────────────────────────────────────────────────────────────┐ │
│   │              시스템 프레임워크 (불변)                           │ │
│   │   SSOT | 라이프사이클 | 신선도 | 관계 | 버전                    │ │
│   ├───────────────────────────────────────────────────────────────┤ │
│   │  GA 보험영업  │  메디코드   │  제조업 A사  │  ...              │ │
│   │  (도메인)     │  (도메인)   │  (도메인)    │                   │ │
│   └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 시스템이 제공하는 것 vs 도메인이 정의하는 것

| 시스템 프레임워크 (공통) | 도메인 설정 (각자) |
|------------------------|-------------------|
| 문서 상태 관리 (DRAFT → ACTIVE → DEPRECATED) | 분류 축 정의 (보험사/상품/문서유형) |
| SSOT 강제 (중복 방지) | 분류 값 목록 (삼성생명, 한화생명...) |
| 신선도 자동 계산 | 문서유형별 갱신 주기 |
| 관계 연결 (상하/좌우/참조) | 기본 관계 규칙 |
| 버전 관리 | - |

### 2.3 사용자 경험 설계

```
┌─────────────────────────────────────────────────────────────────────┐
│                       시스템이 복잡함을 숨긴다                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   사용자가 하는 것:              시스템이 알아서 하는 것:             │
│   ─────────────────              ──────────────────────             │
│                                                                     │
│   1. 파일 업로드                 • 파일명 분석 → 분류 자동 제안       │
│   2. 분류 확인/수정 (드롭다운)   • 중복 검사 → SSOT 자동 처리         │
│   3. 관련 문서 연결 (GUI 클릭)   • 양방향 관계 자동 동기화            │
│   4. 저장                        • 버전/날짜 자동 기록               │
│                                  • 신선도 자동 계산                  │
│                                  • 라이프사이클 자동 관리            │
│                                                                     │
│   "복잡한 건 시스템이, 사용자는 문서 작성과 연결만"                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. 문제 해결 설계

### 3.1 문제 1: 중복/버전 혼란 → SSOT

```
문제 상황:
─────────
📁 같은 문서가 여러 개
├── KB손해_든든어린이_상품요약.pdf
├── KB손해_든든어린이_상품요약_v2.pdf
└── KB손해_든든어린이_상품요약_최신.pdf   ← 어떤 게 유효?

해결 방법:
─────────
┌─────────────────────────────────────────────────────────────────┐
│  분류 경로: KB손해보험 > 든든 어린이보험 > 상품요약본             │
│                                                                 │
│  규칙: 이 경로에 ACTIVE 문서는 하나만 존재 가능                  │
│                                                                 │
│  새 문서 등록 시:                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 시스템: "같은 문서가 이미 있네요"                         │   │
│  │         "기존 문서를 '대체됨'으로 바꾸고 새 걸 등록할게요" │   │
│  │                                                         │   │
│  │         기존: 상품요약본 v1.0 → DEPRECATED               │   │
│  │         신규: 상품요약본 v2.0 → ACTIVE ✓                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  결과: 유효한 문서는 항상 하나 (SSOT)                            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 문제 2: 연관성 단절 → 관계 연결

```
문제 상황:
─────────
시책이 바뀌면 수수료도 바뀌는데... 몰랐음
→ 고객에게 잘못된 수수료 안내

해결 방법:
─────────
┌─────────────────────────────────────────────────────────────────┐
│  세 가지 관계 유형                                               │
│                                                                 │
│  [상하] 부모-자식                                               │
│        보통약관 ──► 특별약관(암), 특별약관(심장)...              │
│        부모 바뀌면 → 자식들 함께 검토 필요 알림                  │
│                                                                 │
│  [좌우] 형제                                                    │
│        시책 ◄──► 수수료 ◄──► 보험료표                           │
│        하나 바뀌면 → 형제들 함께 조회                            │
│                                                                 │
│  [참조] 인용                                                    │
│        판매스크립트 ──► 상품설명서                               │
│        참조 대상 바뀌면 → "인용 문서 갱신 필요" 알림             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 문제 3: 만료 문서 사용 → 신선도 자동화

```
문제 상황:
─────────
6개월 전 시책으로 상담 → 수수료 잘못 안내 → 클레임

해결 방법:
─────────
┌─────────────────────────────────────────────────────────────────┐
│  문서 유형별 유효 기간                                           │
│                                                                 │
│  HOT  (자주 변경)   시책, 수수료       14~30일                   │
│  WARM (가끔 변경)   상품설명서, 가이드  90일                     │
│  COLD (거의 안변경) 약관, 규정          365일                    │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 시간 경과에 따른 상태 변화                               │   │
│  │                                                         │   │
│  │   등록일                        만료일                   │   │
│  │     │                            │                      │   │
│  │     │◄──── FRESH ────►│◄─ WARNING ─►│◄── EXPIRED ──►   │   │
│  │     │      (안전)       │  (주의)    │   (자동 STALE)   │   │
│  │     ├────────70%────────┼────30%─────┤                  │   │
│  │                                                         │   │
│  │   시책 (30일 기준):                                     │   │
│  │   0~21일: FRESH | 22~30일: WARNING | 31일~: STALE       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.4 문제 4: 찾기 어려움 → 체계적 분류

```
문제 상황:
─────────
"KB 든든어린이 시책 어디 있어요?"
→ "김대리가 알텐데... 김대리 퇴사했는데..."

해결 방법:
─────────
┌─────────────────────────────────────────────────────────────────┐
│  3축 분류 체계 (GA 보험영업 예시)                                │
│                                                                 │
│  WHO (누가)     ───► 보험사    삼성생명, 한화생명, KB손해...     │
│  WHAT (무엇)    ───► 상품      종신보험, 암보험, 실손보험...     │
│  WHICH (어떤)   ───► 문서유형  약관, 시책, 수수료, 스크립트...   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  트리 구조로 탐색                                       │   │
│  │                                                         │   │
│  │  📁 KB손해보험                                          │   │
│  │  ├── 📁 든든 어린이보험                                 │   │
│  │  │   ├── 📄 상품요약본                                  │   │
│  │  │   ├── 📄 시책 (2026-02) ✓                           │   │
│  │  │   ├── 📄 수수료체계                                  │   │
│  │  │   └── 📄 보험료표                                    │   │
│  │  └── 📁 무배당 건강보험                                 │   │
│  │      └── ...                                            │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  결과: 누구나 같은 경로로 문서 접근 가능                         │
└─────────────────────────────────────────────────────────────────┘
```
