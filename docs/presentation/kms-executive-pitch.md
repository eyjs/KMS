# 슬라이드 2: 우리가 겪는 문제

## 매일 반복되는 현장의 어려움

### 설계사들이 겪는 일상

1. **"이 시책 아직 유효한가요?"**
   - 지난달 끝난 시책을 고객에게 안내
   - 수수료 오산정, 고객 민원 발생

2. **"이 자료 어디 있었죠?"**
   - 파일이 카톡, 이메일, 공유폴더에 흩어져 있음
   - 같은 자료 찾는데 30분 소요

3. **"A사 기준인가요, B사 기준인가요?"**
   - 12개 보험사, 수십 개 상품
   - 보험사별 기준 혼동으로 청약 반려

4. **"이 약관 최신 버전 맞나요?"**
   - 구버전과 신버전이 섞여 있음
   - 잘못된 정보로 계약 체결

### 결과

- 상담 품질 저하
- 고객 신뢰 하락
- 매출 기회 손실
- 컴플라이언스 리스크

# 슬라이드 3: 문제의 근본 원인

## 왜 이런 일이 반복될까?

### 현재 상황

| 문제 | 현상 | 영향 |
|------|------|------|
| 문서가 흩어져 있음 | 카톡, 이메일, 개인 폴더 | 찾는 데 시간 낭비 |
| 버전 관리 안 됨 | 구버전 신버전 혼재 | 잘못된 정보 사용 |
| 유효 기간 모름 | 만료된 시책 계속 사용 | 수수료 분쟁 |
| 보험사별 구분 없음 | 자료가 뒤섞임 | 보험사 기준 혼동 |
| 담당자가 바뀌면 지식 단절 | 인수인계 어려움 | 노하우 소실 |

### 핵심 원인

**"문서에 이름표가 없다"**

- 이 문서가 어느 보험사 것인지
- 어느 상품에 해당하는지
- 언제까지 유효한지
- 최신 버전이 맞는지

아무도 모릅니다.

# 슬라이드 4: 해결 방향

## 모든 문서에 이름표를 달자

### KMS의 접근법

**문서 체계 관리 시스템**

모든 문서에 3가지 질문에 답할 수 있게 합니다:

1. **어디에 속하나?** (분류)
   - 어느 보험사, 어느 상품, 어떤 종류의 문서인가

2. **지금 유효한가?** (상태)
   - 작성 중인가, 사용 중인가, 만료되었나

3. **함께 볼 문서는?** (관계)
   - 관련된 다른 문서는 무엇인가

### 핵심 원칙

**"같은 자리에 유효한 문서는 딱 하나"**

- 삼성생명 종신보험 상품설명서는 1개만 유효
- 새 버전이 나오면 이전 버전은 자동으로 만료 처리
- 중복 혼란 원천 차단

# 슬라이드 5: 시스템 개요

## KMS가 하는 일

### 1. 문서 저장

모든 문서를 한 곳에 모읍니다.

- PDF, 마크다운, CSV 형식 지원
- 보험사별, 상품별, 문서유형별 분류
- 중앙 집중 관리

### 2. 분류 체계

문서에 이름표를 붙입니다.

- **보험사**: 삼성생명, 한화생명, KB손해보험...
- **상품**: 종신보험, 어린이보험, 실손보험...
- **문서유형**: 시책, 수수료표, 상품설명서, 약관...

### 3. 상태 관리

문서의 생애를 추적합니다.

- **작성 중**: 아직 완성되지 않은 문서
- **사용 중**: 현재 유효한 문서
- **만료됨**: 새 버전으로 대체된 문서

### 4. 관계 관리

문서 간 연결을 만듭니다.

- 상품설명서 → 약관, 수수료표 연결
- 한 문서를 보면 관련 문서 함께 조회

# 슬라이드 6: 사용 시나리오

## 실제 업무에서 어떻게 활용되나

### 시나리오 1: 상담 준비

**Before (현재)**
1. "KB손해보험 어린이보험 자료 어디 있지?"
2. 카톡 검색, 이메일 검색, 공유폴더 뒤지기
3. 30분 후 자료 발견
4. "이게 최신인가?" 확신 없음

**After (KMS 도입 후)**
1. KMS에서 "KB손해보험 > 어린이보험" 클릭
2. 관련 문서 목록 즉시 표시
3. 녹색 표시 = 유효한 문서
4. 1분 내 정확한 자료 확보

### 시나리오 2: 시책 변경

**Before**
1. 새 시책 파일을 카톡으로 공유
2. 일부 설계사는 못 봄
3. 구 시책으로 계속 상담
4. 수수료 분쟁 발생

**After**
1. 새 시책을 KMS에 등록
2. 기존 시책은 자동으로 "만료" 처리
3. 모든 설계사가 같은 정보 확인
4. 분쟁 원천 차단

### 시나리오 3: 신입 교육

**Before**
1. 선배가 일일이 자료 위치 알려줌
2. 선배가 퇴사하면 지식도 사라짐
3. 신입은 다시 혼란

**After**
1. KMS에서 체계적으로 분류된 자료 확인
2. 교육자료, 상품설명서, 스크립트 연결
3. 누가 와도 동일한 품질의 정보 접근

# 슬라이드 7: 신선도 관리

## 문서의 유통기한

### 문서도 유통기한이 있습니다

| 문서 종류 | 유통기한 | 이유 |
|----------|---------|------|
| 시책, 수수료 | 30일 | 수시로 변경 |
| 상품설명서, 스크립트 | 90일 | 분기마다 업데이트 |
| 약관, 교육자료 | 365일 | 연 1회 정도 변경 |

### 시스템이 자동으로 알려줍니다

- **녹색**: 신선함 - 안심하고 사용
- **노란색**: 주의 - 곧 만료, 갱신 준비
- **빨간색**: 만료 - 즉시 갱신 필요

### 효과

- 만료된 자료 사용 방지
- 담당자에게 갱신 알림
- 항상 최신 정보 유지

# 슬라이드 8: 확장 가능한 구조

## GA 보험영업을 넘어서

### 도메인 무관 프레임워크

KMS는 보험영업만을 위한 시스템이 아닙니다.

**같은 구조로 다른 사업에도 적용 가능:**

| 사업 영역 | 분류 체계 | 적용 예시 |
|----------|----------|----------|
| GA 보험영업 | 보험사 × 상품 × 문서유형 | 시책, 수수료표, 약관 |
| 메디코드 | 서비스 × 단계 × 문서유형 | 매뉴얼, 가이드, 계약서 |
| 교육사업 | 과정 × 모듈 × 자료유형 | 교재, 시험, 수료증 |

### 시스템 규칙 (모든 사업 공통)

- 고유 ID 부여
- 상태 관리 (작성 중 → 사용 중 → 만료)
- 신선도 계산
- 관계 연결
- 중복 방지

### 도메인 설정 (사업별 다름)

- 분류 축 구성
- 분류 값 목록
- 신선도 기준

# 슬라이드 16: Q&A

## 예상 질문

### Q1: 기존 시스템과 뭐가 다른가요?

**A**: 기존에는 파일만 저장했습니다. KMS는 파일에 "이름표"를 붙여서 어디에 속하는지, 지금 유효한지, 관련 문서가 뭔지 알 수 있게 합니다.

### Q2: 사용하기 어렵지 않나요?

**A**: 웹 브라우저에서 클릭 몇 번으로 사용합니다. 별도 프로그램 설치 없이, 스마트폰에서도 접근 가능합니다.

### Q3: 보안은 괜찮나요?

**A**: 회사 내부 서버에 저장됩니다. 외부 업체에는 API로만 접근 가능하고, 모든 접근 기록이 남습니다.

### Q4: 외주 업체가 문제 생기면요?

**A**: 데이터는 우리가 100% 소유합니다. API 표준이 있어서 업체 교체가 쉽습니다. 1일 내 다른 업체로 전환 가능합니다.

### Q5: 다른 부서도 쓸 수 있나요?

**A**: 네. 프레임워크 구조라서 GA 보험영업 외에 메디코드, 교육사업 등 다른 사업에도 같은 방식으로 적용 가능합니다.

# 부록: 용어 설명

| 용어 | 설명 |
|------|------|
| KMS | Knowledge Management System, 지식관리시스템 |
| GA | General Agency, 법인보험대리점 |
| 분류체계 | 문서를 정리하는 규칙 (보험사 > 상품 > 문서유형) |
| 라이프사이클 | 문서의 생애 주기 (작성 중 → 사용 중 → 만료) |
| 신선도 | 문서의 유통기한 상태 |
| SSOT | Single Source of Truth, 같은 자리에 유효 문서 1개 원칙 |
| API | 외부 시스템이 데이터를 가져가는 창구 |
| RAG | AI가 문서를 읽고 질문에 답하는 기술 |
