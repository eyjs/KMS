# KMS 사용자 매뉴얼

## 1. 시스템 개요

KMS(Knowledge Management System)는 문서의 저장, 분류, 관계를 체계적으로 관리하는 시스템입니다.

**핵심 기능:**
- 문서 업로드 및 분류 (PDF, Markdown, CSV)
- 도메인 기반 문서 관리
- 문서 간 관계 설정 및 시각화
- 라이프사이클 관리 (임시저장 → 사용중 → 만료)
- 역할 기반 접근 제어

---

## 2. 로그인

1. 브라우저에서 시스템 URL에 접속합니다.
2. 이메일과 비밀번호를 입력합니다.
3. **초기 관리자 계정**: `admin` / `admin`
4. 로그인 후 좌측 사이드바와 대시보드가 표시됩니다.

> 보안을 위해 초기 비밀번호는 반드시 변경해 주세요.

---

## 3. 대시보드

로그인 후 첫 화면이 대시보드입니다.

### 통계 카드
- **전체 문서**: 등록된 문서 총 수 (클릭 시 검색 페이지로 이동)
- **사용중(ACTIVE)**: 현재 유효한 문서 수
- **임시저장(DRAFT)**: 아직 게시되지 않은 문서 수
- **만료(DEPRECATED)**: 더 이상 유효하지 않은 문서 수

### 조치 필요 문서
- 갱신이 필요하거나 유효기간이 임박한 문서 목록
- 클릭하면 해당 문서 상세 페이지로 이동

### 최근 활동
- 최근 업로드, 수정, 상태 변경 등의 이력

---

## 4. 도메인 관리

도메인은 **불변하는 업무 단위**입니다. 조직(본부/부서/팀)이 아닌 업무를 기준으로 설정합니다.

### 도메인 구조
- 트리 형태의 계층 구조 (최대 4단계)
- 예: `문서관리(ROOT)` → `일반` / `경영/관리` / `기술`

### 도메인 생성 (관리자)
1. 좌측 사이드바 → **Admin** → **도메인 관리**
2. **도메인 추가** 버튼 클릭
3. 입력 항목:
   - **이름**: 업무 영역명 (예: 영업, 기술)
   - **상위 도메인**: 소속될 상위 도메인 선택
   - **필수 분류**: 이 도메인의 문서에 반드시 필요한 분류 유형
   - **SSOT 키**: 동일 분류 조합에서 ACTIVE 문서를 1개로 제한하는 기준
4. 코드는 자동 생성됩니다.

### 도메인 수정/삭제
- 수정: 이름, 설명, 필수 분류, SSOT 키 변경 가능
- 삭제: 하위 도메인이나 문서가 있으면 삭제 불가

---

## 5. 분류 유형 관리 (Facet Type)

분류 유형은 문서를 분류하는 **기준의 종류**입니다.

### 기본 제공
- **문서유형(docType)**: 시스템 유형 — 삭제/코드변경 불가

### 분류 유형 추가 (관리자)
1. 좌측 사이드바 → **Admin** → **시스템 설정**
2. **분류 유형 추가** 버튼 클릭
3. 입력 항목:
   - **코드**: 영문 식별자 (예: department, region)
   - **이름**: 한국어 표시명 (예: 부서, 지역)
   - **코드 접두어**: 분류 값의 자동 코드 생성 시 접두어 (예: D → D001, D002)
   - **설명**: 용도 설명

---

## 6. 분류 값 관리 (Facet)

분류 값은 각 분류 유형의 **실제 선택지**입니다.

### 분류 값 추가
1. 도메인 워크스페이스 좌측 패널 → 분류 트리
2. 분류 유형 옆 **+** 버튼 클릭
3. 이름을 입력하면 코드가 자동 생성됩니다.
4. 선택 항목: 신선도 등급(HOT/WARM/COLD), 정렬 순서

### 신선도 등급
| 등급 | 설명 |
|------|------|
| HOT | 자주 갱신되는 문서 (가격표, 공문 등) |
| WARM | 주기적 갱신 (안내서, 비교표 등) |
| COLD | 거의 변경 없음 (계약서, 규정집 등) |

## 7. 문서 업로드

### 허용 파일 형식
| 형식 | 뷰어 |
|------|------|
| PDF | pdf.js 뷰어 |
| Markdown (.md) | marked.js 렌더링 |
| CSV | 테이블 렌더링 |

> Word, PowerPoint, Excel 파일은 지원하지 않습니다.

### 업로드 방법
1. 도메인 워크스페이스에서 **업로드** 버튼 클릭
2. 파일 선택 (최대 50MB)
3. 분류 선택:
   - 도메인의 필수 분류를 모두 선택해야 합니다.
   - 예: 문서유형 → "계약서" 선택
4. 보안 등급 선택 (기본: 사내용)
5. **업로드** 클릭

### SSOT (Single Source of Truth)
- 동일 분류 조합에서 ACTIVE 문서는 1개만 허용됩니다.
- 같은 분류로 새 문서를 ACTIVE로 전환하면, 기존 문서는 자동으로 DEPRECATED 됩니다.

## 8. 문서 검색

### 통합 검색 (Ctrl+K)
1. 상단 사이드바 → **통합 검색** 또는 `Ctrl+K` 단축키
2. 키워드 입력 시 파일명, 문서코드, 분류값에서 검색
3. 검색 이력이 자동 저장되어 다음 검색 시 자동완성

### 필터 옵션
- **도메인**: 특정 업무 영역으로 제한
- **라이프사이클**: 임시저장 / 사용중 / 만료
- **보안 등급**: 공개 / 사내용 / 대외비 / 기밀

### 정렬 옵션
- 최신순 (기본)
- 오래된순
- 파일명순
- 도메인순

### URL 동기화
- 검색 조건이 URL에 반영되어 링크 공유가 가능합니다.

---

## 9. 문서 상세

### 문서 뷰어
- PDF: 페이지 탐색, 확대/축소
- Markdown: 렌더링된 문서 표시
- CSV: 테이블 형태로 표시

### 문서 정보 수정
- **분류 변경**: 분류값 수정 가능
- **보안 등급 변경**: 등급 상향/하향 가능

### 라이프사이클 전환
| 현재 상태 | 전환 가능 | 설명 |
|-----------|----------|------|
| 임시저장 | → 사용중 | 문서를 게시합니다 |
| 사용중 | → 임시저장, 만료 | 게시를 철회하거나 만료 처리합니다 |
| 만료 | (없음) | 종료 상태입니다 |

## 10. 관계 관리

문서 간 관계를 설정하여 연관 문서를 체계적으로 관리합니다.

### 관계 유형
| 관계 | 설명 | 양방향 |
|------|------|--------|
| 상위/하위 | 부모-자식 관계 | O |
| 형제 | 같은 수준의 관련 문서 | O |
| 참조 | 다른 문서를 참조 | X |
| 대체 | 이전 버전을 대체 | X |

### 관계 추가
1. 문서 상세 페이지 → **관계** 탭
2. **관계 추가** 버튼 클릭
3. 대상 문서 검색 및 선택
4. 관계 유형 선택

### 관계 그래프
- 문서 상세 페이지에서 관계 그래프를 시각적으로 확인
- 노드 클릭으로 관련 문서로 이동
- 도메인별 색상 구분

### Document Explorer
- 도메인/분류 기준으로 문서를 트리 형태로 탐색

---

## 11. 사용자 관리 (관리자)

### 사용자 목록
1. 좌측 사이드바 → **Admin** → **사용자 관리**
2. 전체 사용자 목록 확인

### 역할 변경
- 사용자 행에서 역할을 변경할 수 있습니다.
- 관리자는 자신의 역할을 변경할 수 없습니다 (자기보호).

### 계정 비활성화
- 퇴사자 등의 계정을 비활성화하여 로그인을 차단합니다.
- 관리자는 자신의 계정을 비활성화할 수 없습니다.

---

## 12. 권한 체계

### 문서 보안 등급
| 등급 | 설명 |
|------|------|
| 공개(PUBLIC) | 외부업체 포함 누구나 |
| 사내용(INTERNAL) | 직원 이상 |
| 대외비(CONFIDENTIAL) | 검토자 이상 |
| 기밀(SECRET) | 승인자 이상 |

### 사용자 역할
| 역할 | 한국어 | 접근 범위 | 주요 권한 |
|------|--------|----------|----------|
| VIEWER | 조회자 | 공개 문서만 | 문서 조회 |
| EDITOR | 작성자 | 사내용까지 | 문서 작성/수정 |
| REVIEWER | 검토자 | 대외비까지 | 문서 검토/삭제 |
| APPROVER | 승인자 | 기밀까지 | 최종 승인 |
| ADMIN | 관리자 | 전체 | 시스템 설정 |

> 역할은 업무 기반이며, 조직 직급과 무관합니다.

## 13. 피드백

시스템 사용 중 문제 발견이나 개선 제안을 할 수 있습니다.

### 피드백 보내기
1. 화면 우측 하단의 **파란색 원형 버튼** 클릭
2. 유형 선택:
   - **버그 신고**: 오류나 비정상 동작
   - **개선 제안**: 기능 추가나 개선 요청
   - **질문**: 사용 방법 문의
3. 제목과 내용을 입력
4. **전송** 클릭
5. 현재 페이지 URL이 자동으로 첨부됩니다.

### 피드백 관리 (관리자)
1. 좌측 사이드바 → **Admin** → **피드백 관리**
2. 상태별/유형별 필터링
3. 행 클릭하여 상세 확인
4. 상태 변경: 접수 → 처리중 → 해결됨 / 닫힘
5. 관리자 메모 작성
